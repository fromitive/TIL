## 한가지 일만 하도록 변경하는 방법

####  특정 객체가 메서드 호출이 많을 경우

**리팩터링 전**
```
g.doSomething(); // <- 1. 메서드 분리 대상
someMethod(g);
someMethod2(g);
```

`g`의 메서드 호출이이 많이 있을 경우, `return값이 g인 메서드로 분리`한다.

**리팩터링 후**
```
let g = handleG();
someMethod(g);
someMethod2(g);
```

### if 문 분리

**리팩터링 전**
```
for-each {
	if(something) {
		doSomething();
	} else {
		doOtherSomeThing();
	}
}
```

if 문 안에 있는 블럭을 하나의 작업 단위로 분리한다.

**리팩터링 후**

```
for-each{
	doSomethingCondition()
}

doConditionalSomething {
	if (something) {
		doSomeThing();
	}
}
```

## 이해가 안가는 부분

if 문을 왜 한 개의 작업단위로 보냐? 

이해가 안가는 구절
>  five-line code 46page - 임 우리는 함수가 한 가지 일만 해야 한다는 것을 압니다. 무언가를 확인하는 것은 한가지 일입니다. 따라서 함수에 if가 있는 경우 함수의 첫 번째 항목이어야 합니다. 또한 그 후에 아무것도 해서는 안된다는 유일한 것이어야 합니다. 

if의 else까지의 한 단위로 처리한다. 이는 4장 타입 코드 처리에서 살펴볼 수 있다.
